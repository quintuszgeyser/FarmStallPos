First-Time Setup Admin

  1. Open PowerShell and run the QA start script:
    powershell -ExecutionPolicy Bypass -File start-qa.ps1
  2. On first run, all database tables are created automatically and an admin user is seeded.
  3. Trust the HTTPS certificate (see below) — required once per machine.
  4. Open https://127.0.0.1:5000 in Edge or Chrome.
  5. Log in with admin / admin123 and change the password immediately.

Trusting the HTTPS Certificate Admin

The app uses HTTPS with a self-signed certificate. Browsers will show a security warning until you install the certificate into Windows. This is required for the camera barcode scanner to work (browsers block camera access on plain HTTP).

Do this once per machine You only need to install the certificate once. After that, Edge and Chrome trust the app automatically on all future visits.
  1. Right-click the Windows Start button and choose Windows PowerShell (Admin) or Terminal (Admin).
  2. Run:
    powershell -ExecutionPolicy Bypass -File "C:\...\farm_pos_web\install-cert.ps1"
    (replace C:\... with your actual project path)
  3. Restart Edge or Chrome.
  4. Open the app — no certificate warning should appear.

If you see the warning again after regenerating the cert, just re-run install-cert.ps1 as Administrator.

To promote QA changes to production:

  1. Commit and push all changes on the main branch.
  2. Run: powershell -ExecutionPolicy Bypass -File promote.ps1
  3. Type yes to confirm. The script merges main into the production branch and pushes both to GitHub.

Session & Auto-Logout Both

The system tracks login activity automatically for staff time management.

  • Idle 10 minutes: the current activity session closes for time-tracking purposes. The teller stays logged in — no interruption.
  • Idle 2 hours: automatic logout. The teller must log in again when they return.

This means a teller can step away for 20 minutes between customers without being logged out, but will be logged out automatically at end of day.

Processing a Sale Both

Product Dropdown

Click "Select a product…" at the top, scroll or type to find, and click to add.

Search Bar

Type a product name, ID, or barcode. Click a result to add it.

USB / Bluetooth Barcode Scanner

  1. Be on the Teller tab with no text input focused (click an empty area if needed).
  2. Scan. The screen flashes green and the item is added.
  3. If the barcode is not found, a warning toast shows the scanned code.

Camera Scanner

  1. Click Scan to activate the camera. HTTPS is required for camera access.
  2. Point at the barcode. Item is added automatically with a 1.5-second cooldown.
  3. Click Stop to turn off the camera.

Cart Management Both

  • Adjust quantity: use + / − on each cart line.
  • Remove an item: click Remove on the cart line.
  • Undo last scan: click ↩ Undo at the top of the cart.

Weight & Volume Items Both

  1. Add the product. The Weight Entry modal opens automatically.
  2. Enter the quantity and select the unit (g, kg, ml, L, or a package unit).
  3. The live price preview updates as you type.
  4. Click Add to Cart.

Automatic Specials Both

Specials (combo deals) detect and apply automatically every time the cart changes. The system finds the combination that saves the customer the most money. Each product unit counts toward at most one special.

Customising Recipes Both

  1. Add a recipe product to the cart.
  2. Click Customise on that cart line.
  3. Swap ingredients, remove them, or add extras.
  4. A price adjustment is shown based on ingredient cost differences.
  5. Click Add to Cart. The cart line shows a summary of changes.

Checkout Both

  1. Confirm the cart items and total.
  2. Click Checkout.
  3. Stock is deducted (FIFO), kitchen orders are created for prepared items, and the cart clears.
Important Once checked out, only an admin can edit or void the transaction. Double-check the cart before checking out.

Viewing Transactions Teller

Go to the Transactions tab to see your last 5 transactions: ID, date/time, teller, line items, and total. COGS and margin are not shown to tellers.

Flagging a Transaction Both

  1. Find the transaction and click ⚑ Flag.
  2. Enter a note describing the issue.
  3. Click Flag for Admin Review.

Kitchen Queue Both

The Kitchen tab shows pending preparation orders. A red badge shows the count and updates every 30 seconds.

  • Each card shows: product, quantity, ingredient details (swaps/extras highlighted), teller name, live timer (turns red after 10 min).
  • ▲ / ▼ — reprioritise orders.
  • ✓ Done — mark complete (records wait time).
  • ✕ Cancel — cancel with confirmation.
  • Completed Today — toggle history of completed orders.

Each count badge updates automatically as products are added or changed.

Stock Item Cards

Products with trackable stock (stock_item type) show as expandable cards. Click anywhere on a card to expand it and see:

  • All FIFO batches: date received, supplier, quantity bought, quantity left, stock value, cost per unit, sale price per unit
  • Sell packages (pre-packed portions)
  • Edit ✏️ button per batch to correct the purchase date, supplier, or total cost

The action buttons on each card:

Adding a Product Admin

  1. Click + New Product.
  2. Choose the product type:
    • Stock Item — trackable by weight/volume/count with FIFO costing. Use for anything you receive from a supplier and sell (drinks, biltong, cheese).
    • Recipe — made from stock items. Use for prepared items (cappuccino, toasted sandwich).
    • Simple — integer count stock, no FIFO costing. Use for legacy or simple count items.
  3. Fill in the fields and click Add.
Note on pricing The Margin % calculator is only available when editing an existing product — it requires stock batches to calculate a cost basis. For new products, just enter the selling price directly.

Editing & Pricing Admin

  1. Click Edit on a product card.
  2. Make your changes. The Margin % field and Calculate & Set Price button appear above the selling price — enter a margin % and click Calculate to see the suggested price based on current FIFO cost.
  3. Click Update to save.

Archiving & Restoring Admin

Archiving

  1. Click Archive on the product card.
  2. If the product has remaining stock, choose what to do with it:
    • Keep stock — stock remains in the batches, visible from the Archived tab. Use this if you might re-activate the product.
    • Write off all remaining stock — FIFO batches are consumed and an adjustment is logged. Use this when the product is permanently discontinued.
  3. If the product is used in active recipes, choose per recipe: archive it, remove the ingredient, or replace with another ingredient.
  4. Click Archive to confirm.

Restoring

Go to the Archived sub-tab, click Restore on the product. Any cascade-archived recipes that can be restored (all their other ingredients are still active) will be offered for restoration too.

Receiving Stock Admin

  1. Find the product in the Products tab (Single Items or Ingredients sub-tab) and click Receive.
  2. Enter the quantity received. If a purchase package is configured (e.g. 5L carton), you can enter 3 cartons instead of 15 000ml — the system converts automatically.
  3. Enter the total amount paid (R). The cost per unit is calculated and shown live in friendly units (e.g. R8.33/L, not R0.008333/ml).
  4. Optionally select a Supplier.
  5. Click Confirm — Receive Stock.

A new FIFO batch is created. Future sales consume older batches first.

Stocktake (Physical Count) Admin

  1. Click Stocktake on a product card.
  2. Enter the physically counted quantity. Click + Add another container if stock is split across locations.
  3. The preview shows the difference vs. the system quantity (green = match, yellow = loss, blue = surplus).
  4. Enter a reason (required).
  5. Click Apply Stocktake Adjustment.

Write-Off (Spoilage / Damage) Admin

Caution Write-offs permanently remove stock and record a cost loss. Use only for genuinely spoiled or damaged goods.
  1. Click Write Off on a product card.
  2. Enter the quantity and unit. The estimated FIFO cost consumed is shown live.
  3. Enter a reason (required) — e.g., Milk expired 2026-04-25.
  4. Click Confirm Write-Off.

The write-off loss appears on the Stats tab. To correct an error, find the adjustment in the Adjustment History section at the bottom of the Products tab, click ✏️, and correct the quantity.

Setting Up Specials Admin

Go to Products → Specials.

  1. Click + New Special.
  2. Enter a name (shown as toast at the till) and the special price.
  3. Click + Add Product for each product in the deal and set the required quantity.
  4. Check Active to enable it at the till.
  5. Click Save Special.

Suppliers Admin

Go to the Suppliers tab.

Adding a Supplier

  1. Click + New at the top of the supplier list.
  2. Fill in: name (required), phone, email, website, and notes as separate fields.
  3. Click Save. The supplier's detail panel opens automatically.

Viewing Supplier Details

Click any supplier in the list. The right panel shows their contact details (phone, email, website are clickable links) and a list of products ever received from them, including last received date.

Editing a Supplier

With a supplier selected, click Edit in the detail panel. The form loads above. Make changes and click Save.

Purchase Run Admin

A Purchase Run lets you record an entire shopping trip from one supplier in one go — useful when returning from Woolworths, PnP, or a market with a slip.

  1. Select the supplier and click + Purchase Run.
  2. Set the date (defaults to today — change it if you're recording a previous trip).
  3. For each item on your slip, click + Add Item:
    • Select the product from the dropdown — products previously bought from this supplier appear at the top.
    • Enter quantity and unit (only units valid for that product type are shown).
    • Enter the total amount paid for that item (R).
  4. To add a product that doesn't exist yet, click + Create New Product on any line. The full product creation form opens. After saving, the new product is auto-selected on that line.
  5. Click Save Purchase Run. All batches are created and stock is updated in one operation.
Tip The unit dropdown automatically shows only the correct units for each product (kg/L for weight/volume items, unit for count items).

Managing Transactions Admin

Date Range Filter

Use the Start / End date pickers and click Filter, or click a preset (Today, Yesterday, etc.).

Editing a Transaction

  1. Click Edit / Void on a transaction.
  2. Adjust quantities or unit prices in the table. Remove lines with ✕.
  3. Click Save Changes. Original stock is restored and new stock is deducted atomically.

Voiding a Transaction

  1. Click Edit / Void.
  2. Enter a void reason (required).
  3. Click Void Transaction. All stock is fully restored.

Resolving Flags

On a flagged transaction, click ✓ Resolve. The flag badge changes to ✓ Reviewed.

User Management Admin

Go to the Users tab.

Adding a User

  1. Fill in username, password, and role (teller or admin).
  2. Ensure Active is checked.
  3. Click Add User.

Disabling vs Deleting

Prefer disabling (uncheck Active, click Update User) over deleting — it preserves the user's transaction history while preventing login.

Click any employee row to expand a detailed session breakdown showing:

  • Login — when they opened the app
  • Logout — when they logged out or timed out
  • Last Activity — last time they clicked anything (tells you when they stopped working)
  • Duration — length of the session
  • Status — Active (still logged in) or Closed
How session time works Sessions auto-close after 10 minutes of inactivity and a new one opens on the next click — so time is tracked in activity windows, not wall-clock time. A teller with a quiet hour between customers will show multiple short sessions, which is accurate.

Exporting Data Admin

  1. Go to the Stats tab and scroll to the Export section.
  2. Set the start and end dates.
  3. Click Download Sales CSV for transactions, or Products CSV for the product catalogue.
Direct URL /admin/export/transactions?start=YYYY-MM-DD&end=YYYY-MM-DD and /admin/export/products. Admin login required.
