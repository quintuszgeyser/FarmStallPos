Farm Stall POS — User Guide

Version 1.6  ·  App runs at http://127.0.0.1:5000

This guide covers every feature of the Farm Stall POS system. Use the sidebar to jump to any section. Sections are marked with role badges: Teller Admin Both

Login & Logout Both

Logging In

  1. Open http://127.0.0.1:5000 in any browser.
  2. Enter your Username and Password in the login card.
  3. Click Login. You will land on the Teller tab automatically.

Default admin credentials: username admin, password admin123 — change this after first setup.

Logging Out

Click the Logout button (top-right of the screen, or in the user card). The session is immediately cleared and the login screen is shown.

Processing a Sale Both

The Teller tab is the main point-of-sale screen. Items are added to the cart and then checked out as a single transaction.

Adding Items to the Cart

There are four ways to add a product:

1 — Product Dropdown

  1. Click the dropdown labelled "Select a product…" at the top of the Teller tab.
  2. Scroll or type to find the product. The format shown is Product name — R[price].
  3. Click the product. It is added to the cart immediately.

2 — Search Bar

  1. Type a product name, ID number, or barcode number in the search bar.
  2. Matching results appear as a dropdown list showing name and stock level.
  3. Click a result to add it to the cart. The search field clears automatically.

3 — USB / Bluetooth Barcode Scanner

  1. Ensure you are on the Teller tab and no text input field is focused (click an empty area if needed).
  2. Scan the barcode with the scanner. The scanner sends keystrokes ending with Enter — the system captures this automatically.
  3. On a successful match: a beep plays, the screen flashes green, and the item is added to the cart.
  4. If the barcode is not found, a warning toast appears showing the scanned code.

4 — Camera Scanner

  1. Click the Scan button to activate the device camera.
  2. Point the camera at the barcode. The system uses ZXing to detect barcodes in real time.
  3. Once detected, the item is added and a 1.5-second cooldown prevents duplicate scans.
  4. Click Stop to turn off the camera.

Cart Management Both

Adjusting Quantity

For fixed-price (non-weight) items, use the + and − buttons on each cart line. The price updates automatically. The minimum quantity is 1.

Removing an Item

Click the Remove button on the cart line. The item is removed immediately with no confirmation required.

Undo Last Scan

Click the ↩ Undo button at the top of the cart to reverse the most recent add. If the item has qty > 1, qty decreases by 1; otherwise the item is removed.

Tip For items sold by weight (e.g. biltong), quantity adjustment via + / − is not available. To correct a weight, remove the item and re-add it with the correct weight.

Weight & Volume Items Both

Some products are sold by weight (g / kg) or volume (ml / L) — for example, fresh biltong weighed at the counter.

  1. Add the product via scanner, search, or dropdown. The Weight Entry modal opens automatically.
  2. Enter the quantity in the Quantity field.
  3. Select the unit from the dropdown (g, kg, ml, L, or a custom package unit if configured).
  4. The live price preview updates as you type: 250g = R62.50.
  5. Click Add to Cart.

Each weight entry creates a separate line in the cart even if the same product is added multiple times — this allows multiple bags of different weights for the same customer.

Automatic Specials Both

Specials (combo deals) are detected and applied automatically every time the cart changes. You do not need to do anything — the system handles it.

How It Works

  • The system checks all active specials against the cart.
  • It finds the combination that saves the customer the most money.
  • Each product unit is counted toward at most one special — no double-dipping.
  • A green toast appears: "Family Deal ×1 — saving R5.00".
  • The discounted prices are shown on the cart lines and in the total.
Example — overlapping specials Specials: "2× Coke = R30" and "1× Coke + 1× Cream Soda = R25".

Cart: 2× Coke + 1× Cream Soda
The system compares both options and picks the one that saves more. In this case "Coke + Cream Soda" wins because applying it uses one Coke (leaving one at full price) whereas "2× Coke" would leave the Cream Soda at full price and save less overall.

Cart: 3× Coke + 1× Cream Soda
Now both specials can apply without conflict: "Coke + Cream Soda" uses 1 Coke + 1 Cream Soda, and "2× Coke" uses the remaining 2 Cokes. Both discounts are applied simultaneously.

Customising Recipes Both

Recipe products (cappuccinos, toasted sandwiches, etc.) can be customised per customer order.

  1. Add the recipe product to the cart.
  2. Click the Customise button on that cart line.
  3. In the modal, adjust any ingredient:
    • Swap: Change the "Using" dropdown to a different ingredient (e.g. Oat Milk instead of Full Cream Milk). Previously used alternatives appear at the top for quick access.
    • Remove: Click Remove to exclude an ingredient entirely.
    • Extra: Click + Add extra ingredient to add something not normally in the recipe.
  4. A price adjustment is shown at the bottom based on ingredient cost differences.
  5. Click Add to Cart to confirm. The cart line name shows a summary of changes, e.g. Cappuccino (Milk→Oat Milk, +Cinnamon).

The kitchen order sent to the kitchen queue will show all substitutions, removals, and extras highlighted in colour so staff know exactly how to prepare the item.

Checkout Both

  1. Confirm the cart items and total at the bottom of the cart panel.
  2. Click the Checkout button (disabled if cart is empty).
  3. The system records the transaction, deducts stock using FIFO costing, and sends any prepared items to the kitchen queue.
  4. A success message shows the transaction ID and the number of kitchen orders created (if any).
  5. The cart clears automatically, ready for the next customer.
Important Once checkout is complete, only an admin can edit or void the transaction. Confirm the cart is correct before checking out.

Viewing Transactions Teller

Go to the Transactions tab to see your recent sales. As a teller, you see your last 5 transactions only.

Each transaction card shows:

  • Transaction ID (first 8 characters)
  • Date and time
  • Your name (teller badge)
  • Each line item: product × qty @ price = subtotal
  • Total amount

Flagging a Transaction Both

If something is wrong with a transaction (wrong product, price dispute, customer complaint), flag it for admin review without altering the transaction data.

  1. Find the transaction in the Transactions tab.
  2. Click the ⚑ Flag button on the transaction card.
  3. Enter a clear note explaining the issue — e.g., "Customer returned Coke, said it was warm".
  4. Click Flag for Admin Review.

The transaction is marked with a yellow ⚑ Flagged badge. Admins can see all flagged transactions and will resolve them.

Kitchen Queue Both

The Kitchen tab shows all pending preparation orders. A red badge on the tab shows the count of pending orders and updates every 30 seconds.

Working the Queue

  • Each card shows: product name, quantity, ingredient details (including swaps and extras), the teller who placed the order, and a live timer.
  • The timer turns red after 10 minutes to indicate urgency.
  • Use ▲ / ▼ buttons to reprioritise orders.
  • Click ✓ Done when the order is ready. The completion time and wait time are recorded.
  • Click ✕ Cancel to cancel an order (with confirmation).

Viewing History

Click Completed Today to toggle a list of completed and cancelled orders for the current day, including wait times.

Managing Transactions Admin

Date Range Filter

  1. Go to the Transactions tab.
  2. Use the Start and End date pickers, then click Filter. Click Today for a quick today-only view.

Editing a Transaction

  1. Find the transaction and click Edit / Void.
  2. In the modal, adjust quantities or unit prices directly in the table. Remove a line with the ✕ button.
  3. The total recalculates live as you type.
  4. Click Save Changes. The system restores original stock and re-deducts based on the new values.
Stock impact Saving changes restores all original stock and then deducts the new quantities. There is a brief moment of reconciliation — do not refresh mid-save.

Voiding a Transaction

  1. Click Edit / Void on the transaction.
  2. Enter a void reason in the text field (required).
  3. Click Void Transaction.
  4. The transaction is marked voided, all stock is fully restored, and the reason is recorded for the audit trail.

Resolving Flags

On a flagged transaction, click ✓ Resolve. The badge changes from ⚑ Flagged to ✓ Reviewed and the note box turns grey.

Products Management Admin

Go to the Products tab. Use the sub-tabs to navigate: For Sale, Ingredients, Archived, Specials.

Adding a New Product

  1. Click + New Product.
  2. Choose the product type:
    • Stock Item — anything with trackable stock (drinks, biltong, produce).
    • Recipe — made from stock items (cappuccino, toasted sandwich).
    • Simple — legacy count-based stock (no FIFO costing).
  3. Fill in the required fields (see tables below).
  4. Click Add Product.

Editing a Product

Click Edit on any product card. Make your changes and click Update Product.

Price Calculator

  1. In the product editor, click Calculate Price.
  2. Enter a target margin % (or click a quick button: 20%, 30%, 40%…).
  3. Click Calculate & Set Price. The breakdown shows ingredient/batch costs.
  4. Click Use This Price to apply the suggested price.

Archiving & Restoring

Archiving a product that is used in recipes

When you archive an ingredient used in recipes, the system shows you each affected recipe with three options:

  1. Archive the recipe too (cascade).
  2. Remove the ingredient from the recipe.
  3. Replace with a different ingredient.

Choose the appropriate action per recipe, then click Archive.

To restore a product, go to the Archived sub-tab, click Restore, and optionally restore any recipes that were cascade-archived with it.

Setting Up Specials Admin

Go to Products → Specials. Specials are combo deals that automatically apply at the till.

Creating a Special

  1. Click + New Special.
  2. Enter a name (shown in toast notifications at the till) — e.g., Family Deal.
  3. Enter the special price — the total price when all required products are in the cart.
  4. In the Required Products table, click + Add Product for each product in the deal. Set the required quantity per product.
  5. Check Active to make the special available at the till immediately.
  6. Click Save Special.
Example To create "2× Coke + 1× Chips = R30": name it "Braai Special", set price to R30, add Coke qty 2 and Chips qty 1.

Editing / Deleting a Special

Click Edit on any special card. Modify the fields and click Save Special, or click Delete to remove it permanently.

How the Till Handles Conflicting Specials

If two specials share a product (e.g. Coke appears in both "2× Coke" and "Coke + Cream Soda"), the system automatically finds the combination that gives the customer the best saving. Each unit is counted toward at most one special. See the Automatic Specials section for a detailed example.

Receiving Stock Admin

Go to the Stock tab to manage all stock-item products.

  1. Find the product and click the green Receive button.
  2. Enter the quantity received. If a purchase package is configured (e.g. 5L carton), you can enter 3 cartons instead of 15 000ml — the system converts automatically and shows the base-unit equivalent below.
  3. Enter the total amount paid (R). The cost per base unit is calculated and shown live.
  4. Optionally select a Supplier. Click + next to the dropdown to add a new supplier inline.
  5. Click Confirm — Receive Stock.

A new FIFO batch is created with its own cost per unit. Future sales will consume this batch after older batches are exhausted.

Stocktake (Physical Count) Admin

  1. Click the yellow Stocktake button on a product in the Stock tab.
  2. Enter the physically counted quantity. For stock split across locations, click + Add another container for each location. The running total is shown.
  3. A preview shows the difference vs. the system quantity:
    • Green: Matches — no adjustment needed.
    • Yellow: System will deduct the unexplained loss.
    • Blue: System will add the surplus found.
  4. Enter a reason (required) — e.g., Monthly stocktake.
  5. Click Apply Stocktake Adjustment.

Write-Off (Spoilage / Damage) Admin

Caution Write-offs permanently remove stock and record a cost loss. Use only for genuinely spoiled or damaged goods.
  1. Click the red Write Off button on a product in the Stock tab.
  2. Enter the quantity to write off and select the unit.
  3. The estimated cost consumed from FIFO batches is shown live.
  4. Enter a reason (required) — e.g., Milk expired 2026-04-25.
  5. Click Confirm Write-Off.

The write-off loss appears on the Stats tab in the Write-off Loss card.

If you entered the wrong quantity, find the adjustment in the Adjustment History section of the Stock tab, click ✏️ Edit, and correct the quantity.

Suppliers Admin

Adding a Supplier

  1. Go to the Suppliers tab.
  2. Fill in the form on the right: name (required), phone, email, website, notes.
  3. Click Save.

Editing / Deleting a Supplier

Click Edit on a supplier card in the list. The form loads on the right. Make changes and click Save, or click Delete to remove (historical purchase batches retain the supplier name).

User Management Admin

Go to the Users tab.

Adding a User

  1. Fill in Username, Password, and select a Role (teller or admin).
  2. Ensure Active is checked.
  3. Click Add User.

Editing a User

  1. Click Edit on the user in the list — the form loads on the right.
  2. Change the role, active status, or password (leave password blank to keep the existing one).
  3. Click Update User.
Disabling a user Instead of deleting, uncheck Active and click Update User. The user cannot log in but their transaction history is preserved.

Deleting a User

Click Edit on the user, then click Delete User. A confirmation dialog appears. The action is permanent — consider disabling instead.

Stats & Reports Admin

Selecting a Date Range

Use the From / To date pickers, or click a preset button (Today, Yesterday, This Week, This Month, This Year), then click Load.

Click the tabs below the summary cards to switch between:

Exporting Data Admin

Transactions CSV

  1. Go to the Stats tab, scroll to the Export section.
  2. Set the start and end dates.
  3. Click Download Sales CSV. The file downloads immediately.

Products CSV

Click the Products CSV link (also in the Stats export section) for a download of all products including IDs, names, barcodes, prices, and stock levels.

Direct URL export You can also download CSVs directly: /admin/export/transactions?start=YYYY-MM-DD&end=YYYY-MM-DD and /admin/export/products. Admin login required.
